# 双鱼直播 — 财富等级 & 魅力等级获取说明

## 数据来源

等级数据存储在 DataBinding 绑定的 ViewModel 中：

```
类名: com.sybl.voiceroom.data.entity.model.RoomOnlineItemViewModel
位置: ItemRoomOnlineUserListBindingImpl → 父类字段 mViewModel
```

### 关键字段（均为 `ObservableField<String>`）

| 字段 | 类型 | 含义 | 示例值 |
|------|------|------|--------|
| `wealthImg` | `ObservableField<String>` | 财富等级图片 URL | `https://res-eo.shuangyuxingqiu.com/等级/财富/55.png` |
| `charmImg` | `ObservableField<String>` | 魅力等级图片 URL | `https://res-eo.shuangyuxingqiu.com/等级/魅力/60.png` |
| `gender` | `ObservableField<Integer>` | 性别（Android 资源 ID） | `2131690251` (男) |
| `nickName` | `ObservableField<String>` | 用户昵称 | `ʜ.树` |
| `userId` | `ObservableField<String>` | 用户 ID | `22228722` |
| `userCode` | `ObservableField<String>` | 用户展示 ID | `ID:22228722` |

### 性别字段说明

`gender` 的值是 Android drawable 资源 ID，不是字符串，需要映射：

| 资源 ID | 资源名 | 性别 |
|---------|--------|------|
| `2131690251` | `res_ic_boy_circle` | 男 (male) |
| `2131690314` | `res_ic_girl_circle` | 女 (female) |

### 等级数字提取规则

URL 格式固定，等级数字就是 URL 最后一段路径的文件名（去掉 `.png`）：

```
财富: .../等级/财富/{等级数字}.png  → 取 {等级数字}
魅力: .../等级/魅力/{等级数字}.png  → 取 {等级数字}
```

**Java 提取方式：**
```java
String url = "https://res-eo.shuangyuxingqiu.com/等级/财富/55.png";
String fileName = url.substring(url.lastIndexOf('/') + 1);  // "55.png"
String level = fileName.replace(".png", "");                  // "55"
int levelNum = Integer.parseInt(level);                       // 55
```

## 技术实现路径

### 为什么不能用无障碍 API？

等级图标（`iv_wealth` / `iv_charm`）是 `AppCompatImageView`，等级数字直接渲染在 Bitmap 图片上，
没有 `text`、`contentDescription`、`tag` 等可读属性，UIAutomator 和 AccessibilityService 都无法读取。

### LSPosed Hook 路径

1. **Hook 时机**：`Activity.onCreate` 后（确保应用类已加载）
2. **Hook 目标**：`ItemRoomOnlineUserListBindingImpl.executeBindings()`
3. **数据读取**：
   - 通过反射获取绑定对象的 `mViewModel` 字段
   - 调用 `ObservableField.get()` 读取 `wealthImg` / `charmImg` 的 URL
   - 从 URL 解析等级数字
4. **数据回传**：通过 `dump_levels` 广播命令请求，模块解析后以 `ACTION_LEVEL_DATA_REPORT` 广播返回

### 调试日志验证

```bash
# 触发 dump_levels 命令
adb shell am broadcast -a com.oodbye.shuangyulspmodule.ACTION_ENGINE_COMMAND \
  --es engine_command dump_levels --el engine_command_seq 0

# 查看等级数据输出
adb logcat -v time -d -s SYLspModule | grep -a "LEVEL" | head -20
```

## 数据回传广播格式

**Action**: `com.oodbye.shuangyulspmodule.ACTION_LEVEL_DATA_REPORT`

**Extras**:
| Key | 类型 | 说明 |
|-----|------|------|
| `level_data_json` | `String` | JSON 数组，每个元素包含一个用户的等级信息 |

**JSON 格式示例**：
```json
[
  {
    "userId": "22228722",

    "genderResId": 2131690251,
    "wealthImg": "https://...财富/55.png",
    "charmImg": "https://...魅力/60.png"
  }
]
```
